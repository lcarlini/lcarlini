Loyalty Banking Account

Overview & Challenge

The platform managed participant registration and current accounts (credit, debit, transfer, refund, balance, statement, and integrations). Growth exposed performance and scalability limits, heavy business logic in procedures, and a fragile manual process for splitting large files — often risking SLA breaches.

Development Process

  • Replaced FTP uploads with Azure-based batch processing for better scalability.
  • Built an API integrated with ServiceNow for client interaction, full file tracking, and pre-validation of invalid files.
  • Treated each file line as a Service Bus message so Azure Functions could scale processing; completion events notified dependent systems.
  • Added scheduling for multi-million-record files, including overnight runs for long workloads.

Achievements & Impact

Improved UX, resolved architectural issues, achieved true large-file scalability, and contributed to a 27% increase in company revenue after the improvements.

Point Expiration System

Overview & Challenge

The point expiration flow had transaction issues that caused financial loss. Participants can use balance until 23:59 on the last valid day, which made correct cutoff handling critical.

Development Process

Introduced a Redis lock one day before expiration to force additional validation. After expiration is confirmed, the lock is removed and expiring transactions become Service Bus messages processed by an Azure Function for scalable execution.

Protected revenue that was previously being lost and improved the reliability and performance of expiration processing.

How I work

Engineering principles I stand by

The mindset behind the code — what keeps systems trustworthy and teams moving fast.

/ 01

Simplicity is a feature

The best architecture is the simplest one that solves the problem well. Complexity is a cost you pay forever.

/ 02

Design for change

Requirements evolve. Clear boundaries, clean domains, and loose coupling keep systems adaptable, not brittle.

/ 03

Reliability is non-negotiable

Testing, monitoring, and observability aren't extras — they're how software earns trust in production.

/ 04

Business value leads

Every technical decision serves an outcome. I translate business goals into systems that actually deliver them.

/ 05

Lift the whole team

Mentoring, reviews, and shared standards make everyone better — durable results come from strong teams.

/ 06

Never stop learning

From cloud-native to AI-assisted engineering, staying curious is how I keep delivering at the edge.

/ 07

Honesty

Clear communication about risks, trade-offs, and progress — saying what is true even when it is uncomfortable, so teams can decide well.

/ 08

Commitment

Owning outcomes end to end — following through on promises, finishing what we start, and staying accountable when things get hard.
